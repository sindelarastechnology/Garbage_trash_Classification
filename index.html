2.1 Bagaimana Citra Dibaca oleh Python

# Ada 2 library utama untuk membaca citra di Python: # 1. OpenCV (cv2) - format BGR img_bgr = cv2.imread('path/to/image.jpg') # 2. PIL (Python Imaging Library) - format RGB from PIL import Image img_pil = Image.open('path/to/image.jpg') # Program kita menggunakan OpenCV, jadi perlu konversi BGR → RGB: img_rgb = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2RGB)

🔍 Mengapa Perlu Konversi BGR ke RGB?

  • OpenCV (cv2.imread): Membaca citra dalam format BGR (Blue-Green-Red)
  • Matplotlib & Neural Network: Mengharapkan format RGB (Red-Green-Blue)
  • Tanpa konversi: Warna akan terbalik (merah jadi biru, biru jadi merah)

2.2 Format dan Ukuran Citra

2.3 Representasi Citra (RGB/Grayscale/Pixel Array)

A. Citra RGB (3 Channel)

Citra RGB terdiri dari 3 channel warna (Red, Green, Blue):

ImageRGB = [R, G, B]
Dimensi = (Height, Width, 3)

Setiap pixel memiliki 3 nilai (R, G, B), masing-masing berkisar 0-255.

# Contoh representasi pixel RGB: img_rgb[:5, :5, 0] # Channel Red (5×5 pixel pertama) # Output: array nilai intensitas merah di pojok kiri atas # [[201 201 201 200 200] # [201 201 201 200 200] # [201 201 201 200 200] # [201 201 201 200 200] # [201 201 201 200 200]] # Visualisasi channel RGB secara terpisah: fig, axes = plt.subplots(1, 4, figsize=(16, 4)) axes[0].imshow(img_rgb) # Citra asli RGB axes[1].imshow(img_rgb[:, :, 0], cmap='Reds') # Channel Red axes[2].imshow(img_rgb[:, :, 1], cmap='Greens') # Channel Green axes[3].imshow(img_rgb[:, :, 2], cmap='Blues') # Channel Blue

B. Citra Grayscale (1 Channel)

Citra grayscale hanya memiliki 1 channel (intensitas abu-abu):

Gray = 0.299 × R + 0.587 × G + 0.114 × B
Dimensi = (Height, Width)

Konversi RGB ke Grayscale menggunakan weighted average (mata manusia lebih sensitif terhadap hijau).

# Konversi RGB ke Grayscale: img_gray = cv2.cvtColor(img_rgb, cv2.COLOR_RGB2GRAY) # Sekarang shape menjadi (Height, Width) tanpa channel print(img_gray.shape) # Output: (224, 224)

C. Pixel Array (Numpy Array)

Dalam Python, citra direpresentasikan sebagai Numpy Array:

  • ✅ Akses cepat: Operasi vektor yang efisien
  • ✅ Manipulasi mudah: Operasi aritmatika langsung pada pixel
  • ✅ Kompatibilitas: Bekerja dengan library ML seperti TensorFlow

3.1 Proses 1: RESIZE (Penyeragaman Ukuran)

IMG_SIZE = (224, 224) img_resized = cv2.resize(img_rgb, IMG_SIZE) print(f'Ukuran asli : {img_rgb.shape[:2]}') # (256, 256) print(f'Ukuran resize : {img_resized.shape[:2]}') # (224, 224)
🔹 Fungsi:
Menyeragamkan ukuran semua citra menjadi 224×224 pixel agar konsisten untuk input model CNN.
🔹 Alasan Penggunaan:
  • Konsistensi Input: Model CNN memerlukan input dengan ukuran tetap
  • Arsitektur MobileNetV2: Dirancang untuk input 224×224 px
  • Efisiensi Komputasi: Ukuran lebih kecil = training lebih cepat
  • Standar ImageNet: Ukuran 224×224 adalah standar pada transfer learning

3.2 Proses 2: GRAYSCALE (Konversi Abu-abu)

img_gray = cv2.cvtColor(img_resized, cv2.COLOR_RGB2GRAY) print(f'Shape RGB : {img_resized.shape}') # (224, 224, 3) print(f'Shape Grayscale : {img_gray.shape}') # (224, 224)
🔹 Fungsi:
Mengubah citra berwarna (3 channel RGB) menjadi citra abu-abu (1 channel).
🔹 Alasan Penggunaan:
  • Reduksi Dimensi: Mengurangi 3 channel menjadi 1
  • Fokus pada Struktur: Analisis bentuk tanpa gangguan warna
  • Preprocessing Edge Detection: Canny memerlukan grayscale

3.3 Proses 3: NORMALISASI PIXEL

img_normalized = img_resized.astype('float32') / 255.0
🔹 Fungsi:
Mengubah nilai pixel dari rentang 0-255 menjadi 0.0-1.0.
🔹 Alasan Penggunaan:
  • Stabilitas Training: Gradient descent lebih stabil
  • Konvergensi Cepat: Model belajar lebih efisien
  • Standar Neural Network: Pre-trained model butuh input 0-1

3.4 Proses 4: GAUSSIAN BLUR (Filtering)

img_blur = cv2.GaussianBlur(img_gray, (5, 5), 0)
🔹 Fungsi:
Menghaluskan citra dan mengurangi noise menggunakan kernel Gaussian 5×5.
🔹 Alasan Penggunaan:
  • Noise Reduction: Menghilangkan gangguan kecil pada citra
  • Smoothing: Membuat transisi antar pixel lebih halus
  • Preprocessing Segmentasi: Hasil thresholding lebih baik

3.5 Proses 5: SEGMENTASI (Otsu Thresholding)

_, img_thresh = cv2.threshold(img_blur, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
🔹 Fungsi:
Memisahkan objek dari latar belakang menggunakan metode Otsu's thresholding.
🔹 Alasan Penggunaan:
  • Pemisahan Objek-Background: Otomatis menentukan threshold optimal
  • Otsu Algorithm: Mencari threshold yang memaksimalkan variance antar kelas
  • Ekstraksi Objek: Memudahkan analisis bentuk objek sampah

3.6 Proses 6: EDGE DETECTION (Canny)

img_edges = cv2.Canny(img_gray, threshold1=50, threshold2=150)
🔹 Fungsi:
Mendeteksi tepi/kontur objek dalam citra menggunakan algoritma Canny.
🔹 Alasan Penggunaan:
  • Deteksi Bentuk: Mengidentifikasi batas objek dengan akurat
  • Analisis Struktur: Membantu CNN mengenali pola geometris sampah
  • Multi-stage Algorithm: Noise reduction → Gradient → Non-max suppression → Hysteresis

3.7 Proses 7: EKSTRAKSI FITUR (Histogram Warna)

hist_r = cv2.calcHist([img_resized], [0], None, [256], [0, 256]) hist_g = cv2.calcHist([img_resized], [1], None, [256], [0, 256]) hist_b = cv2.calcHist([img_resized], [2], None, [256], [0, 256])
🔹 Fungsi:
Mengekstrak distribusi warna dari citra sebagai representasi fitur.
🔹 Alasan Penggunaan:
  • Fitur Warna: Distribusi warna unik untuk tiap jenis sampah
  • Histogram 256 bins: Merepresentasikan frekuensi setiap intensitas warna
  • Karakteristik Material: Kaca, plastik, logam punya pola warna berbeda

🤖 Soal 4: Aplikasi Pengolahan Citra (CNN - MobileNetV2)

Bobot: 15 | Topik: Algoritma Machine Learning untuk Citra

4.1 Nama Algoritma yang Digunakan

Convolutional Neural Network (CNN) dengan Transfer Learning

Arsitektur Spesifik: MobileNetV2 (pre-trained on ImageNet)

  • 🔸 Base Model: MobileNetV2 (dari TensorFlow/Keras)
  • 🔸 Teknik: Transfer Learning
  • 🔸 Custom Head: GlobalAveragePooling2D + Dense layers
  • 🔸 Output: 6 kelas (cardboard, glass, metal, paper, plastic, trash)

4.2 Apa itu CNN (Convolutional Neural Network)?

CNN adalah jenis neural network yang dirancang khusus untuk memproses data berbentuk grid seperti gambar.


Perbedaan dengan Neural Network Biasa:

  • ✅ Preservasi Struktur Spasial: CNN mempertahankan hubungan antar pixel
  • ✅ Parameter Sharing: Filter yang sama digunakan di seluruh gambar
  • ✅ Translation Invariance: Dapat mengenali objek di posisi berbeda

Komponen Utama CNN:

Layer Fungsi Operasi
Convolutional Layer Ekstraksi fitur lokal Filter/kernel mengekstrak pola (edges, textures)
Activation (ReLU) Non-linearity ReLU(x) = max(0, x) - menghilangkan nilai negatif
Pooling Layer Downsampling Mengurangi dimensi, ambil nilai max/average
Fully Connected Klasifikasi Dense layer untuk prediksi akhir

4.3 Cara Kerja CNN Secara Hierarkis

# Proses hierarkis CNN: Input Image (224×224×3) ↓ [Layer 1] Deteksi Edges (tepi, garis) ↓ [Layer 2] Deteksi Textures (pola tekstur) ↓ [Layer 3] Deteksi Shapes (bentuk objek) ↓ [Layer 4] Deteksi Parts (bagian objek) ↓ [Layer 5] Deteksi Objects (objek lengkap) ↓ Output: Probabilitas 6 kelas

📌 Konsep "Feature Hierarchy":

  • Layer Awal: Mendeteksi fitur sederhana (edges, corners)
  • Layer Tengah: Menggabungkan edges menjadi tekstur dan pola
  • Layer Akhir: Mengenali objek kompleks (bentuk botol, kaleng, dll)

4.4 Transfer Learning: Mengapa Menggunakan MobileNetV2?

🔹 Apa itu Transfer Learning?

Transfer Learning adalah teknik menggunakan model yang sudah dilatih pada dataset besar (ImageNet) dan mengadaptasinya untuk task baru (garbage classification).


Model Pre-trained (ImageNet 1.4M images) → Fine-tune → Garbage Classification (13.9K images)
🔹 Keuntungan Transfer Learning:
  • ⚡ Training Lebih Cepat: Tidak perlu train dari nol
  • 📊 Akurasi Lebih Tinggi: Model sudah belajar fitur visual umum
  • 💾 Data Requirement Lebih Kecil: Cukup 13K gambar vs 1M+
  • 🎯 Generalisasi Lebih Baik: Menghindari overfitting

4.5 Arsitektur MobileNetV2

MobileNetV2 adalah arsitektur CNN yang dioptimalkan untuk perangkat mobile/embedded dengan resource terbatas.

Karakteristik MobileNetV2:

Fitur Deskripsi Keuntungan
Depthwise Separable Convolution Memisahkan spatial dan channel-wise convolution Mengurangi komputasi hingga 8-9x
Inverted Residuals Expand → Depthwise → Project Efisiensi memori dan akurasi tinggi
Linear Bottlenecks Tidak ada ReLU di projection layer Mempertahankan informasi penting
Total Parameters ~3.5 juta (base model) Ringan, cocok untuk deployment

4.6 Arsitektur Model dalam Program Kita

# Struktur model lengkap: Input: (224, 224, 3) ↓ BASE MODEL: MobileNetV2 (pre-trained ImageNet) - 53 Convolutional Layers - Depthwise Separable Convolutions - Inverted Residual Blocks - Output shape: (7, 7, 1280) ↓ CUSTOM CLASSIFICATION HEAD: - GlobalAveragePooling2D → (1280,) - Dense(256, activation='relu') → (256,) - Dropout(0.5) → regularisasi - Dense(6, activation='softmax') → (6,) ↓ Output: [P(cardboard), P(glass), P(metal), P(paper), P(plastic), P(trash)]

✅ Penjelasan Setiap Layer:

  • GlobalAveragePooling2D: Mengambil rata-rata setiap feature map (7×7×1280 → 1280)
  • Dense(256, relu): Fully connected layer dengan 256 neuron untuk representasi fitur tingkat tinggi
  • Dropout(0.5): Randomly "mematikan" 50% neuron saat training untuk mencegah overfitting
  • Dense(6, softmax): Output layer dengan softmax untuk probabilitas 6 kelas (total = 1.0)

4.7 Fungsi Algoritma Terhadap Citra

🔹 Bagaimana CNN Memproses Citra Sampah?
  1. Input Preprocessing: Citra di-resize ke 224×224 dan dinormalisasi (0-1)
  2. Feature Extraction (MobileNetV2):
    • Layer awal: Deteksi edges dan tekstur material (kaca = smooth, kardus = rough)
    • Layer tengah: Pola warna (plastik = colorful, metal = shiny)
    • Layer akhir: Bentuk objek (botol, kaleng, kertas)
  3. Feature Aggregation (GlobalAveragePooling): Rangkum semua fitur spasial
  4. Classification (Dense Layers): Mapping fitur ke probabilitas kelas
  5. Prediction: Pilih kelas dengan probabilitas tertinggi

📌 Penjelasan Metrik:
  • Precision: Dari semua prediksi kelas X, berapa persen yang benar?
    Precision = TP / (TP + FP)
  • Recall: Dari semua data kelas X yang sebenarnya, berapa persen yang terdeteksi?
    Recall = TP / (TP + FN)
  • F1-Score: Harmonic mean dari Precision dan Recall
    F1 = 2 × (Precision × Recall) / (Precision + Recall)

Berikut 15 pertanyaan yang kemungkinan besar ditanyakan dosen beserta jawaban lengkapnya:

1. Mengapa memilih dataset Garbage Classification ini?

Jawaban:

Saya memilih dataset ini karena relevan dengan isu lingkungan yang sedang urgent, yaitu pengelolaan sampah. Dataset ini memiliki karakteristik yang baik untuk pembelajaran:

  • Dataset seimbang (13.901 gambar tersebar merata di 6 kelas)

  • Aplikasi praktis jelas: sistem pemilahan sampah otomatis
  • Kompleksitas cukup untuk demonstrasi teknik pengolahan citra
2. Jelaskan proses konversi BGR ke RGB, mengapa penting?

Jawaban:

OpenCV membaca citra dalam format BGR (Blue-Green-Red), sedangkan Matplotlib dan neural network mengharapkan RGB (Red-Green-Blue). Konversi dilakukan dengan:

img_rgb = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2RGB)

Pentingnya: Tanpa konversi, warna akan terbalik (merah jadi biru, biru jadi merah), yang akan membuat model belajar pola warna yang salah dan menurunkan akurasi.

3. Dari 7 preprocessing yang diterapkan, mana yang paling critical dan mengapa?

Jawaban:

Yang paling critical adalah Resize dan Normalisasi:

  • Resize (224×224): MUTLAK diperlukan karena CNN memerlukan input dengan dimensi tetap. Tanpa ini, model tidak bisa dilatih.
  • Normalisasi (0-1): Sangat critical untuk stabilitas training. Dengan nilai 0-255, gradient bisa explode/vanish. Normalisasi membuat convergence 3-5x lebih cepat.

Sementara teknik lain (Gaussian Blur, Edge Detection) bersifat opsional untuk analisis tambahan.

4. Bagaimana Gaussian Blur mengurangi noise? Jelaskan secara matematis!

Jawaban:

Gaussian Blur menggunakan kernel 5×5 dengan distribusi Gaussian:

G(x, y) = (1 / 2πσ²) × e^(-(x²+y²) / 2σ²)

Kernel ini di-convolve dengan citra, memberikan bobot lebih tinggi pada pixel pusat dan menurun ke tepi. Efeknya:

  • Noise frekuensi tinggi (pixel acak) ter-smooth karena di-average dengan tetangganya
  • Transisi tajam (edges) sedikit blur namun tetap terdeteksi
  • Hasil: citra lebih halus, thresholding dan edge detection lebih akurat
5. Mengapa menggunakan Otsu thresholding, bukan manual threshold?

Jawaban:

Otsu's method secara otomatis mencari threshold optimal yang memaksimalkan variance antar kelas (background vs foreground).

Keunggulan:

  • Adaptive: Tidak perlu manual tuning threshold untuk tiap gambar
  • Robust: Bekerja baik meski lighting berbeda-beda
  • Optimal: Secara matematis mencari titik pemisahan terbaik

Manual threshold (misal: 128) tidak fleksibel - bisa bagus untuk satu gambar tapi buruk untuk gambar lain.

6. Jelaskan perbedaan CNN dengan Neural Network biasa!

Jawaban:

Aspek NN Biasa (Fully Connected) CNN
Struktur Input Flatten 1D (224×224×3 = 150K param per neuron) Preserve 2D (kernel 3×3 = 27 param)
Parameter Sangat banyak (prone overfitting) Jauh lebih sedikit (parameter sharing)
Spatial Info Hilang saat flatten Dipertahankan (pixel tetangga penting)
Translation Invariance Tidak ada Ada (konvolusi di seluruh gambar)

Kesimpulan: CNN dirancang khusus untuk data grid (gambar) dan jauh lebih efisien.

7. Mengapa menggunakan Transfer Learning daripada training from scratch?

Jawaban:

Transfer Learning memanfaatkan MobileNetV2 yang sudah dilatih pada ImageNet (1.4 juta gambar, 1000 kelas).

Keuntungan konkret:

  • Waktu: Training hanya 2 jam vs 2-3 hari from scratch
  • Data: Cukup 13K gambar vs butuh 100K+ from scratch
  • Akurasi: 91.79% vs ~70-75% from scratch dengan data sama
  • Generalisasi: Pre-trained model sudah belajar fitur visual universal (edges, textures)

Kita tinggal fine-tune layer atas untuk task spesifik (garbage classification).

8. Jelaskan cara kerja Dropout(0.5) dalam mencegah overfitting!

Jawaban:

Dropout dengan rate 0.5 berarti setiap neuron punya probabilitas 50% "dimatikan" (set to 0) saat training.

Mekanisme mencegah overfitting:

  • Forced Redundancy: Network tidak boleh terlalu bergantung pada neuron tertentu
  • Ensemble Effect: Setiap batch melatih sub-network berbeda → seperti ensemble model
  • Feature Co-adaptation: Neuron dipaksa belajar fitur yang robust, tidak hanya menghafal

Saat inference: Semua neuron aktif, tapi output di-scale dengan 0.5 untuk kompensasi.

9. Mengapa validation accuracy (91.79%) tidak jauh berbeda dari training accuracy (92.03%)?

Jawaban:

Gap kecil (hanya 0.24%) menunjukkan model tidak overfitting dan memiliki generalisasi baik.

Alasan gap kecil:

  • Transfer Learning: Pre-trained weight sudah general
  • Dropout & Augmentation: Regularisasi mencegah overfitting
  • Data Cukup: 11K training samples cukup untuk 2.5M parameter yang di-train
  • Early Stopping: Training berhenti sebelum overfitting terjadi

Jika gap >5%, itu indikasi overfitting. Gap kami sangat sehat untuk production deployment.

10. Kelas mana yang paling sulit dibedakan dan mengapa?

Jawaban:

Dari confusion matrix dan F1-scores, kelas paling sulit adalah:

  • Glass vs Metal (15 kesalahan): Kedua material punya karakteristik reflektif tinggi dan shiny appearance yang mirip.
  • Trash (Recall 0.86, terendah): Kategori "trash" sangat heterogen (organik, campuran) sehingga tidak punya pola visual konsisten.

Solusi: Tambahkan sub-kategori untuk trash (organik, non-organik) atau gunakan ensemble model.

11. Jelaskan proses backpropagation dalam training CNN!

Jawaban:

Forward Pass: Input → Conv layers → Pooling → Dense → Output (predictions)

Loss Calculation: Categorical Crossentropy = -Σ(y_true × log(y_pred))

Backward Pass (Backpropagation):

  1. Hitung gradient loss terhadap output layer: ∂L/∂output
  2. Propagate mundur menggunakan chain rule: ∂L/∂weight = ∂L/∂output × ∂output/∂weight
  3. Update weights menggunakan Adam: weight_new = weight_old - learning_rate × gradient

Proses ini diulang untuk setiap batch (32 gambar) hingga model converge.

12. Apa fungsi GlobalAveragePooling2D dibanding Flatten?

Jawaban:

Aspek Flatten GlobalAveragePooling2D
Output Shape (7×7×1280) = 62,720 values (1280,) = 1,280 values
Parameter Dense 62,720 × 256 = 16M param 1,280 × 256 = 327K param
Overfitting Risk Tinggi (banyak param) Rendah (regularisasi built-in)
Translation Invariance Tidak ada Ada (average semua posisi)

Kesimpulan: GAP mengurangi parameter drastis dan lebih robust terhadap posisi objek.

13. Bagaimana cara meningkatkan akurasi model lebih tinggi lagi?

Jawaban:

Strategi untuk mencapai 95%+ akurasi:

  1. Fine-tuning: Unfreeze 20 layer terakhir MobileNetV2, train ulang dengan LR=0.0001
  2. Ensemble Model: Combine MobileNetV2 + EfficientNetB0 + ResNet50 (voting)
  3. Advanced Augmentation: Mixup, CutMix, AutoAugment
  4. Focal Loss: Fokus pada hard examples (kelas yang sering salah)
  5. Test-Time Augmentation: Prediksi pada 5 augmented versions, ambil rata-rata
14. Apa limitasi dari model ini untuk aplikasi real-world?

Jawaban:

Limitasi teknis:

  • Single Object: Model hanya handle 1 sampah per gambar, tidak bisa multiple objects
  • Controlled Environment: Training data foto close-up, bisa gagal pada foto jarak jauh
  • 6 Kelas Saja: Tidak handle sampah kategori lain (e-waste, hazardous)
  • Lighting Dependency: Performa turun pada kondisi gelap/sangat terang

Solusi: Gunakan object detection (YOLO) untuk multiple objects, tambahkan kelas baru, augmentasi brightness.

15. Jelaskan trade-off MobileNetV2 vs ResNet atau EfficientNet!

Jawaban:

Model Parameters Accuracy (ImageNet) Inference Speed Use Case
MobileNetV2 3.5M ~72% Sangat cepat Mobile/embedded
ResNet50 25M ~76% Sedang Server/cloud
EfficientNetB0 5M ~77% Cepat Balance

Tahap 1: Resize. Gambar diubah ukurannya dari 256x256 menjadi 224x224 pixel menggunakan cv2.resize. Ukuran 224x224 adalah ukuran standar input untuk MobileNetV2.

Tahap 2: Grayscale. Citra RGB 3 channel diubah menjadi 1 channel abu-abu menggunakan cv2.cvtColor dengan parameter RGB2GRAY. Ini membantu reduksi dimensi dan analisis tekstur.

Tahap 3: Normalisasi. Nilai pixel dibagi 255 sehingga berada di rentang 0.0 hingga 1.0. Ini penting untuk stabilitas training neural network.

Tahap 4: Gaussian Blur. Kernel Gaussian 5x5 diterapkan untuk menghaluskan citra dan mengurangi noise.

Tahap 5: Segmentasi Otsu. Metode thresholding otomatis yang memisahkan objek dari latar belakang dengan mencari threshold optimal.

Tahap 6: Canny Edge Detection. Mendeteksi tepi objek menggunakan dua threshold, 50 dan 150.

Tahap 7: Ekstraksi Fitur Histogram. Histogram warna RGB 256 bins diekstrak untuk merepresentasikan distribusi warna."
